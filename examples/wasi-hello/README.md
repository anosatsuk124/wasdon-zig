# wasi-hello

Minimal WASI Preview 1 hello-world fixture for `wasdon-zig`. The module
calls `fd_write(stdout, "Hello, Udon!\n")` and then `proc_exit(0)`. That
is the entire program — exactly the MVP surface defined in
[`docs/spec_wasi_preview_1.md`](../../docs/spec_wasi_preview_1.md).

## Producer choice: hand-written WAT

The producer is a single hand-written `wasi_hello.wat`, converted with
`wat2wasm`. Rust `wasm32-wasip1` was deliberately **not** used here, even
though the spec recommends exercising a real toolchain. Rationale:

1. **Determinism.** A `wasm32-wasip1` Rust binary's import set depends
   on which functions wasi-libc's static initialisers happen to
   reference at the linker's whim. The spec itself flags this in
   §10: "fd_write and proc_exit are the only entries wasi-libc's
   _start actually calls in a trivial program. The deferred-stub list
   still has to exist because wasi-libc's static initialisers reference
   (but do not call) several of the others." That dependence makes a
   `cargo build` fixture brittle to regenerate across rustc versions.
2. **Minimality.** The goal of this example is to demonstrate the MVP
   path — `fd_write` + `proc_exit` and nothing else. Hand-written WAT
   guarantees an import list of exactly two functions; verified below.
3. **Toolchain footprint.** Regenerating the `.wasm` needs only
   `wat2wasm` (a 1-binary install via `brew install wabt`), versus a
   pinned `rustup target add wasm32-wasip1` plus the load-bearing
   `Cargo.toml` profile knobs (`opt-level = "z"`, `lto = true`,
   `codegen-units = 1`, `panic = "abort"`) that the `__udon_meta_ptr`
   const-fold contract requires.

The `wasm32-wasip1` route stays exercised end-to-end by larger fixtures
that the translator gains later (e.g. once Task 3 lands, anyone wanting
a real-toolchain regression test can drop a sibling `wasi-hello-rs/`
example following the recipe in
[`docs/producer_guide.md`](../../docs/producer_guide.md) §8.1).

## Build

```sh
# Install wabt once (provides wat2wasm + wasm-objdump).
brew install wabt   # macOS; on Linux: apt-get install wabt

# Regenerate the fixture from this directory.
cd examples/wasi-hello
wat2wasm wasi_hello.wat -o wasi_hello.wasm
```

Output path: `examples/wasi-hello/wasi_hello.wasm` (about 476 bytes).

## Translate (future, once Task 3 lands)

```sh
zig build run -- examples/wasi-hello/wasi_hello.wasm
```

The translator does not yet recognise `wasi_snapshot_preview1` imports;
attempting this today fails before WASI lowering is wired into
`src/translator/lower_import.zig`. See
[`docs/spec_wasi_preview_1.md`](../../docs/spec_wasi_preview_1.md) §3.1
for the dispatcher design that needs to land first.

## Verification (what `wasm-objdump -x wasi_hello.wasm` reports)

```
Import[2]:
 - func[0] sig=0 <wasi_snapshot_preview1.fd_write> <- wasi_snapshot_preview1.fd_write
 - func[1] sig=1 <wasi_snapshot_preview1.proc_exit> <- wasi_snapshot_preview1.proc_exit
Global[2]:
 - global[0] i32 mutable=0 <__udon_meta_ptr> - init i32=64
 - global[1] i32 mutable=0 <__udon_meta_len> - init i32=234
Export[4]:
 - memory[0] -> "memory"
 - global[0] -> "__udon_meta_ptr"
 - global[1] -> "__udon_meta_len"
 - func[2] <_start> -> "_start"
```

This satisfies every contract the producer guide requires:

- Exactly two WASI imports, both in the MVP set (§2.1 of the WASI spec).
- `__udon_meta_ptr` / `__udon_meta_len` are exported immutable globals
  whose init expression is a single `i32.const` — form (1) of the
  const-eval contract in `CLAUDE.md`.
- The 234-byte meta range `[64, 298)` lies wholly inside data
  segment[2] `(offset=64, size=234)` — no
  `MetaSpansMultipleSegments`.
- `_start` is exported as a nullary function and is mapped to the Udon
  `_start` event via `__udon_meta.functions.start`.
- `behaviour.name` is `WasiHello`.

## Memory layout

| Address | Bytes | Purpose |
|---|---|---|
| `0x0000` | 13 | UTF-8 message `"Hello, Udon!\n"` |
| `0x0010` | 8 | iovec `{ buf=0x0, buf_len=13 }` |
| `0x0020` | 4 | `nwritten` scratch (written by `fd_write`) |
| `0x0040` | 234 | `__udon_meta` JSON blob |

One 64 KiB page is plenty for everything above; `memory.maxPages = 1`
in the meta blob matches.

## Gotchas for future contributors regenerating the fixture

- Keep the meta JSON inside a **single** `(data ...)` segment. Splitting
  it across two segments returns `MetaSpansMultipleSegments` even if
  the address range is contiguous.
- If you change the JSON, update the `i32.const 234` literal in
  `__udon_meta_len`. There is no compile-time way to derive it from the
  data segment in raw WAT.
- Do not switch the global init from `i32.const` to a `global.get` of
  another global unless that other global is **module-defined and not
  imported** — `evalExportedI32` allows exactly one `global.get` hop and
  the source must itself be `i32.const`.
- Keep the import list to `fd_write` + `proc_exit` only. Any extra
  WASI import added here will translate as a `nosys` stub today and
  silently weaken the test.
